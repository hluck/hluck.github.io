Java字符串

String vs StringBuffer

1.String保存的是字符串常量，里面的值不能更改，每次String类的更新实际上就是更改地址，效率较低，//字符串放在：private final char value[];

2.StringBuffer保存的是字符串变量，里面的值可以更改，每次StringBuffer的更新实际上可以更新内容，不用每次更新地址，效率较高。//char[] value; //这个放在堆

3.StringBuffer代表可变的字符序列，可以对字符串内容进行增删。

4.StringBuffer很多方法和String相同，但StringBuffer是可变长的。

5.StringBuffer是一个容器。

String vs StringBuffer vs StringBuilder

1.StringBuilder 和 StringBuffer 非常类似，均代表可变的字符序列，且方法也一样。

2.String：不可变字符序列，效率低但复用率高。

3.StringBuffer:可变字符序列，效率较高（增删），线程安全

4.StringBuilder：可变字符序列，效率最高，线程不安全
